Mobile App Logo Integration Guide

This guide explains how to integrate the Taskium logo into your mobile application to ensure it displays correctly on all devices.

Logo File Options

We've created several versions of the logo to ensure compatibility across all devices:

/mobile-logo.svg
/taskium-logo.svg
/icons/taskium-logo.svg

Important: SVG files may not display correctly in all mobile WebView environments. We recommend using our embedded SVG solution or the CSS fallback option for maximum compatibility.

Option 1: Embedded SVG (Most Compatible)

This approach directly embeds the SVG code in your HTML, ensuring the logo displays on all devices without requiring an external file:

<div class="logo-container">
  <!-- Embedded SVG logo for maximum compatibility -->
  <svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 512 512" fill="none">
    <!-- Outer circle with white stroke -->
    <circle cx="256" cy="256" r="240" stroke="white" stroke-width="32" fill="transparent"/>
    
    <!-- Main checkmark inside circle -->
    <path d="M335 170 L230 320 L170 250" stroke="white" stroke-width="40" stroke-linecap="round" stroke-linejoin="round" fill="transparent"/>
    
    <!-- Partial checkmark/bracket shape on left -->
    <path d="M210 320 C210 320 185 310 180 285 C175 260 190 240 200 230" stroke="white" stroke-width="25" stroke-linecap="round" fill="transparent"/>
    
    <!-- Dot at top of bracket -->
    <circle cx="205" cy="225" r="12" fill="white"/>
    
    <!-- Dot at bottom of bracket -->
    <circle cx="180" cy="285" r="12" fill="white"/>
  </svg>
</div>

Option 3: External SVG with Fallbacks

This approach loads an external SVG file but includes multiple fallbacks if it fails:

<img src="/mobile-logo.svg" alt="Taskium Logo" class="logo" id="appLogo" onerror="handleLogoError()">

<script>
  function handleLogoError() {
    console.log('Logo failed to load, trying alternatives...');
    
    // Define all possible logo paths
    const logoPaths = [
      '/taskium-logo.svg',
      '/icons/taskium-logo.svg',
      '/icons/custom-checkmark-logo.svg'
    ];
    
    const logoElement = document.getElementById('appLogo');
    let currentPathIndex = 0;
    
    // Try loading the next logo in the list
    function tryNextLogo() {
      if (currentPathIndex < logoPaths.length) {
        const nextPath = logoPaths[currentPathIndex];
        console.log(`Trying logo at: ${nextPath}`);
        logoElement.src = nextPath;
        currentPathIndex++;
      }
    }
    
    // Handle errors for alternative logos
    logoElement.onerror = function() {
      tryNextLogo();
    };
    
    // Start trying alternatives
    tryNextLogo();
  }
</script>

View a working example here: /taskium-test.html

WebView Implementation in Android App

For your Android WebView app, you should modify your MainActivity.java to enable embedded SVG support:

// In your configureWebView() method
WebSettings webSettings = webView.getSettings();
webSettings.setJavaScriptEnabled(true);
webSettings.setDomStorageEnabled(true);
webSettings.setCacheMode(WebSettings.LOAD_DEFAULT);
webSettings.setAllowFileAccess(true);
webSettings.setMixedContentMode(WebSettings.MIXED_CONTENT_ALWAYS_ALLOW);

// Enable SVG support
webSettings.setLoadWithOverviewMode(true);
webSettings.setUseWideViewPort(true);

Implementation Steps

  1. Update your WebView settings as shown above to ensure SVG compatibility.
  2. Modify your welcome page HTML to use the embedded SVG approach (Option 1).
  3. Add a CSS fallback (Option 2) as a backup in case of rendering issues.
  4. Test on multiple devices to ensure the logo displays correctly.
Note: If you're still experiencing issues with the logo display in your mobile app, consider bundling the logo directly in your app's assets folder and loading it from there instead of from the web server.
